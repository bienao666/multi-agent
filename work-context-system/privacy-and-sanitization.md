# Privacy And Sanitization

`.work-context/` 很可能包含会议纪要、聊天记录、邮件、客户信息、项目资料和人物画像。把这些材料交给 AI 之前，必须先判断是否需要脱敏。

## 必须脱敏的内容

- 客户真实名称、供应商真实名称、合作方真实名称。
- 真实姓名、手机号、邮箱、地址、身份证件、账号 ID。
- 合同金额、报价、成本、利润、预算、采购信息。
- 内部系统地址、账号、密码、Token、密钥、Cookie。
- 尚未公开的产品计划、商业策略、销售线索、投标信息。
- 人事评价、绩效、薪资、离职、冲突、投诉等敏感信息。
- 任何公司制度要求不得外传的内容。

## 推荐替换方式

| 原始类型 | 替换方式 |
| --- | --- |
| 真实人名 | `人物A`、`直属领导`、`客户决策人`、`协作方A` |
| 客户名 | `客户A`、`重点客户`、`外部合作方` |
| 金额 | `约 X 万`、`预算区间 A`、`金额已脱敏` |
| 系统地址 | `内部系统A`、`管理后台` |
| 项目代号 | `项目A`、`核心项目` |
| 邮箱/手机号 | `联系方式已脱敏` |
| 密钥/Token | `凭证已删除` |

## 人物画像边界

人物画像只记录工作协作信息，不写人格判断。

可以记录：

- 角色和职责。
- 决策关注点。
- 表达偏好。
- 常问问题。
- 对项目的立场。
- 历史反馈。

不要记录：

- 情绪化评价。
- 私生活信息。
- 与工作无关的猜测。
- 侮辱性或不可验证描述。

## 输入 AI 前检查清单

把材料交给 AI 前，检查：

- 是否包含真实姓名或联系方式。
- 是否包含客户、合同、报价、预算等敏感信息。
- 是否包含内部系统地址或凭证。
- 是否包含公司未公开策略。
- 是否包含不必要的人事评价。
- 是否只保留完成任务所需的最小上下文。

## 推荐提示词

```text
请先对下面材料进行脱敏，不改变事实含义。
要求：
1. 替换真实人名、客户名、联系方式、金额、系统地址和凭证。
2. 保留角色、关系、决策关注点、事件经过和后续动作。
3. 删除与当前任务无关的敏感细节。
4. 输出脱敏后的版本，并列出你做了哪些类型的替换。
```
